Manual del Usuario v2.1

MMM.config es una suite de simulación estocástica diseñada para optimizar sistemas de colas. Utiliza WebGL para renderizado en tiempo real y algoritmos basados en eventos discretos para calcular la eficiencia operativa.

info
Requisito: Para simulaciones con más de 100 entidades simultáneas, se recomienda un navegador basado en Chromium (Chrome, Edge, Brave) con aceleración de hardware activada.

Configuración Básica

Antes de iniciar, debes definir los parámetros de entrada en el panel lateral:

  • Lambda (λ): Tasa media de llegadas (clientes por hora).
  • Mu (μ): Tasa media de servicio por servidor (clientes por hora).
  • Servidores (s): Cantidad de nodos de atención activos.
// Ejemplo de configuración estable
λ = 40; μ = 15; s = 3;
// Utilización ρ = 40 / (3*15) = 0.88 (88%)

1. Modelo de Ingeniería

Basado en la teoría de colas clásica M/M/s. Asume colas infinitas, población infinita y disciplina FIFO (First In, First Out).

Ideal para: Dimensionamiento inicial de infraestructura y cálculo de costos base.

2. Modelo Conductual

Introduce variables psicológicas humanas. Los clientes tienen un "umbral de paciencia" aleatorio.

  • Reneging (Abandono) El cliente se retira de la cola después de esperar un tiempo $t > Paciencia$.
  • Balking (Rechazo) El cliente ve la cola muy larga al llegar y decide no entrar al sistema.

3. Modelo Experimental En Desarrollo

science

Este módulo implementará Redes de Jackson (Jackson Networks), permitiendo conectar múltiples sistemas de colas en serie o paralelo.

Referencia API (Local)

El motor expone sus datos a través del objeto global STATE.engine en la consola del navegador.

// Obtener estadísticas actuales en tiempo real
const currentStats = STATE.engine.getStats();
console.log(currentStats);
/* Output:
{
  "atendidos": 45,
  "esperaTotal": 120.5,
  "utilization": 0.85,
  "overtime": 0
}
*/
